# 搜索工具使用规范（所有子技能共享）

本文件定义了 Finance Suite 所有子技能在使用搜索工具时必须遵循的规范。

---

## 一、搜索工具优先级

1. **Tavily Search**（主引擎）：支持 finance/news/general 三种 topic，支持 advanced 深度搜索
2. **Brave Web/News Search**（备引擎）：新闻搜索更快，Tavily 不可用时自动降级
3. **Tavily Extract**：从已知URL提取详细内容（如：财报原文、研报PDF页面）

## 二、数据完整性规则（铁律）

### 规则1：来源标注
每一条从搜索结果获取的事实性数据，必须标注：
```
数据来源：[标题](URL)，检索日期：YYYY-MM-DD
```

### 规则2：数据缺失处理
搜索不到的数据，必须明确标注，严禁编造：
```
[数据暂不可用：截至YYYY-MM-DD未在公开来源中检索到]
```

### 规则3：冲突数据处理
当两个来源的数据冲突时，同时呈现两个来源，不自行裁决：
```
来源A（XX网）显示为XX亿元，来源B（XX网）显示为XX亿元，两者存在差异。
```

### 规则4：时效标注（强制）
所有搜索获取的数据必须标注**原始数据的产生时间**（而非检索时间），格式如下：

**情况A：数据时间明确（如财报、公告）**
```
2025年三季报数据显示：营收XX亿元（+XX%）
```

**情况B：数据时间与当前存在时差（超过3个月）**
必须用醒目方式提示时效差距：
```
截至2025年4月数据：XX只ETF上市，规模XX亿元。
注：以上为2025年4月统计，当前（2026年3月）实际数据可能已发生变化，请以最新官方披露为准。
```

**情况C：数据时间不明确**
```
[数据发布日期不详，引用需谨慎] XX数据显示...
```

**核心原则**：
- 来源日期 ≠ 数据日期 ≠ 检索日期，三者必须区分
- 超过3个月的数据必须标注"可能已过时"
- 绝不让读者误以为旧数据是当前最新数据
- HTML报告中对非实时数据使用视觉提示（如灰色斜体或标签）

## 三、三级降级策略

### 第一级：数据充分
搜索成功，多个来源交叉印证。正常输出完整分析，附来源引用。

### 第二级：数据有限
部分搜索失败或结果不足。输出分析时：
- 有数据支撑的部分正常输出
- 数据不足的部分标注：`以下判断基于有限搜索结果，建议进一步核实`
- 可补充模型训练知识，但必须标注：`基于模型训练数据（非实时），仅供参考`

### 第三级：搜索失败
所有搜索工具均不可用。在输出开头显著提示：
```
注意：实时数据获取失败。以下内容基于模型训练知识，可能不反映最新情况。
建议稍后重试以获取实时数据。
```

## 四、搜索查询优化技巧

1. **中文A股**：查询词加入"东方财富"、"同花顺"等提高中文源命中率
2. **时间控制**：财报用 `--range month`，新闻用 `--range week`，突发用 `--range day`
3. **多轮搜索**：第一轮粗搜确定方向，第二轮精搜补充细节
4. **英文补充**：港美股可加英文ticker辅助搜索
5. **查询长度**：控制在400字符以内，过长会降低搜索质量
